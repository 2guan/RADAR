## 填写示例：新增模块

### 1. 基本信息

- 需求标题：新增“版本风险台账”模块
- 需求类型：新增模块
- 涉及模块或拟新增模块名称：版本风险台账
- 提出人 / 部门：项目管理部
- 优先级：中

### 2. 背景与目标

- 现在有什么问题或业务机会？当前版本风险靠线下表格收集，风险、责任人和处理进展无法与投产点及工作项关联。
- 希望通过本次需求达到什么效果？在 RADAR 内集中登记、跟踪并查看本期投产风险，便于例会跟踪和投产评审。

### 3. 主要使用方式

- 使用人：项目经理、需求负责人、开发负责人、测试负责人、管理员。
- 主要操作：新建风险；关联投产点和需求/工单；指定责任人和计划解决日期；更新处理状态；查看或导出本期未关闭风险。
- 新模块位置与关联：放在“版本概览”下方菜单；关联投产点、需求、工单和投产申请。

### 4. 功能要求

1. 支持新增、编辑、查询、关闭和导出风险记录。
2. 每条风险记录包含风险标题、描述、等级、责任人、计划解决日期和处理状态。
3. 支持关联本期的需求、工单或投产申请，并可从版本概览查看关联风险。
4. 支持按投产点、状态、等级和责任人筛选。

### 5. 结果与验收

1. 项目经理可以按当前投产点新增和查询风险。
2. 未关闭的高风险记录在版本概览中可见。
3. 风险关闭时必须填写处理结果和实际关闭日期。
4. 导出结果与页面筛选结果一致。
5. 普通用户只能维护本人负责或被授权范围内的风险；管理员可维护全部记录。

- 风险与限制：需确认风险是否需要进入投产审批的会签检查项。
