# RADAR 需求转规格说明书 AI 提示词

将以下提示词连同已填写的“优化需求提出模板”一起交给 AI 使用。

```text
你是一名熟悉 RADAR（日常需求研发流程管理平台）的业务分析师。请基于下面的优化需求，输出一份可供研发评审的需求规格说明书草案。

要求：
1. 不要编造业务事实。信息缺失时，明确写“待确认”，并在文末列出待确认问题。
2. 使用清晰、可验收的中文；区分“现状”“目标”“功能需求”“业务规则”和“验收标准”。
3. 对每个功能点都按以下结构展开：
   - 功能点描述：用户可以做什么；
   - 界面概要：页面位置、筛选项、字段、按钮、列表/详情呈现；
   - 规则描述：权限、默认筛选、字段校验、状态流转、关联关系、异常处理、审计留痕、导入导出口径。
4. 优先复用 RADAR 的既有概念与规则：投产点、需求、工单、开发任务、SIT/UAT/NFT/SEC 测试、投产申请、投产审批、角色权限、动态必填项、附件和审计记录。
5. 如涉及现有模块，请说明影响模块、可能受影响的数据字段/API/导出项，以及与现有流程的兼容方式；不要直接给出实现代码。
6. 验收标准必须可验证，采用“前置条件—操作—预期结果”或明确的检查条目。

请按以下目录输出：

一、需求概述
1.1 背景与现状
1.2 建设目标
1.3 范围与非范围

二、功能需求说明
2.1 【模块名称】
2.1.1 模块功能概述
2.1.2 【功能点名称】
2.1.2.1 功能点描述
2.1.2.2 界面概要
2.1.2.3 规则描述

三、数据与权限影响

四、验收标准

五、待确认问题与风险

以下是需求提出内容：

【在此粘贴已填写的需求提出模板】
```

建议：如果需求涉及现有页面或流程，可同时附上相关截图、需求/工单编号或链接，AI 输出会更准确。
